## Create Order

Place a new trading order with various order types and execution options.

### Endpoint
```http
POST /api/v1/orders
```

### Parameters

<ParamField header="X-API-Key" type="string" required>
  API key for authentication. Format: `your_api_key_here`
</ParamField>

### Request Body

<ParamField body="symbol" type="string" required>
  Trading instrument symbol (e.g., "OIL-BTC")
</ParamField>

<ParamField body="instType" type="string" required>
  Instrument type: "perpetual"
</ParamField>

<ParamField body="side" type="string" required>
  Order side: "buy" or "sell"
</ParamField>

<ParamField body="orderType" type="string" required>
  Order type: "market", "limit", "stop", "stopLimit"
</ParamField>

<ParamField body="qty" type="string" required>
  Order quantity as decimal string
</ParamField>

<ParamField body="px" type="string">
  Order price as decimal string (required for limit and stopLimit orders)
</ParamField>

<ParamField body="triggerPx" type="string">
  Stop trigger price as decimal string (required for stop and stopLimit orders)
</ParamField>

<ParamField body="timeInForce" type="string" default="gtc">
  Time in force: "gtc" (Good Till Canceled), "ioc" (Immediate or Cancel), "gtd" (Good Till Date)
</ParamField>

<ParamField body="unit" type="string" default="btc">
  Price units: "sats" or "btc"
</ParamField>

<ParamField body="cancelAt" type="integer">
  Cancel timestamp in milliseconds (for GTD orders)
</ParamField>

<ParamField body="clientOrderId" type="string">
  Client-specified order identifier for tracking
</ParamField>

```python Python
import requests

headers = {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
}

# Market buy order
order_data = {
    'symbol': 'OIL-BTC',
    'instType': 'perpetual',
    'orderType': 'market',
    'side': 'buy',
    'qty': '100',
    'unit': 'sats'
}

response = requests.post(
    'https://api.roxom.com/api/v1/orders',
    headers=headers,
    json=order_data
)

order = response.json()
print(f"Order ID: {order['data']['orderId']}")
```

```javascript JavaScript
const orderData = {
  symbol: 'OIL-BTC',

  body: JSON.stringify(orderData)
});

const order = await response.json();
console.log('Order ID:', order.data.orderId);
```

## Order Validation

Before placing orders, ensure they meet symbol requirements:

<AccordionGroup>
  <Accordion title="Price Validation">
    **For LIMIT and STOP_LIMIT orders:**
    - Price must be within min/max range
    - Price must be divisible by tick size
    - For BUY orders: price ≤ current ask × 1.1
    - For SELL orders: price ≥ current bid × 0.9
  </Accordion>

  <Accordion title="Quantity Validation">
    **For all orders:**
    - Quantity must be within min/max range
    - Quantity must be divisible by step size
    - Notional value (price × quantity) must exceed minimum
  </Accordion>

  <Accordion title="Balance Requirements">
    **For order placement:**
    - BUY orders: require sufficient quote asset balance
    - SELL orders: require sufficient base asset balance
    - Margin orders: require sufficient margin available
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Order Size Strategy">
    - **Small Orders**: Execute faster with less market impact
    - **Large Orders**: Consider breaking into smaller chunks
    - **Iceberg Orders**: Hide large order sizes from the market
  </Accordion>

  <Accordion title="Price Selection">
    - **Limit Orders**: Set competitive prices for better fill rates
    - **Market Orders**: Use for immediate execution, accept slippage
    - **Stop Orders**: Set appropriate buffer from current price
  </Accordion>

  <Accordion title="Risk Management">
    - Always set stop losses for positions
    - Use position sizing appropriate for your risk tolerance
    - Monitor order status and market conditions
  </Accordion>
</AccordionGroup>
