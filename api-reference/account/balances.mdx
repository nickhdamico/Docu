## Get Account Balances

Retrieve current balances for all assets in your account.

### Endpoint
```http
GET /api/v1/accounts/balance
```

### Parameters

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_KEY`
</ParamField>

<ParamField query="asset" type="string">
  Filter by specific asset (e.g., "BTC")
</ParamField>

<ParamField query="non_zero_only" type="boolean" default="false">
  Only return assets with non-zero balances
</ParamField>

### Example Request

<RequestExample>
```bash Account Balance
curl -X GET "https://api.roxom.com/api/v1/accounts/balance" \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.roxom.com/api/v1/accounts/balance',
    headers=headers
)

balance_data = response.json()
print(f"Balance: {balance_data['data']['balance']}")
print(f"Updated at: {balance_data['data']['updated_at']}")
```


## Get Specific Asset Balance

Retrieve balance information for a specific asset.

### Endpoint
```http
GET /v1/account/balances/{asset}
```

### Parameters

<ParamField path="asset" type="string" required>
  Asset symbol (e.g., "BTC", "USD")
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_KEY`
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X GET "https://api.roxom.com/v1/account/balances/BTC" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```python Python
import requests

def get_asset_balance(asset):
    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }
    
    response = requests.get(
        f'https://api.roxom.com/v1/account/balances/{asset}',
        headers=headers
    )
    
    return response.json()

btc_balance = get_asset_balance('BTC')
print(f"BTC Balance: {btc_balance['data']['available']}")
```
</RequestExample>

## Balance Breakdown

Understanding your balance components:

<AccordionGroup>
  <Accordion title="Total Balance">
    **Definition**: The complete amount of an asset in your account
    
    **Formula**: `Total Balance = Available Balance + Locked Balance`
    
    **Use Case**: Portfolio valuation and net worth calculation
  </Accordion>

  <Accordion title="Available Balance">
    **Definition**: The amount available for trading and withdrawals
    
    **Characteristics**:
    - Can be used to place new orders
    - Available for immediate withdrawal
    - Not tied up in pending transactions
    
    **Use Case**: Determine how much you can trade or withdraw
  </Accordion>

  <Accordion title="Locked Balance">
    **Definition**: Balance currently reserved and unavailable for use
    
    **Common Reasons**:
    - Open limit orders
    - Pending withdrawals
    - Margin requirements
    - Settlement periods
    
    **Release**: Locked balance returns to available when:
    - Orders are canceled or filled
    - Withdrawals complete
    - Settlements finalize
  </Accordion>
</AccordionGroup>

## Balance Updates

Balances are updated in real-time when:

- **Orders execute**: Buying/selling assets
- **Orders placed**: Funds locked for open orders
- **Orders canceled**: Locked funds returned to available
- **Deposits received**: New funds added to account
- **Withdrawals processed**: Funds removed from account
- **Fees charged**: Trading or withdrawal fees deducted

## Real-time Balance Monitoring

For real-time balance updates, use the WebSocket API:

```javascript
// Subscribe to account updates
ws.send(JSON.stringify({
  method: 'subscribe',
  params: {
    channel: 'account'
  },
  id: 1
}));

// Receive balance updates
ws.onmessage = function(event) {
  const data = JSON.parse(event.data);
  if (data.channel === 'account' && data.data.update_type === 'balance_change') {
    console.log(`${data.data.asset} balance updated: ${data.data.available}`);
  }
};
```

## Balance Validation

Before trading, always check available balance:

```python
def can_place_order(symbol, side, quantity, price):
    """Check if account has sufficient balance for order."""
    
    base_asset, quote_asset = symbol.split('-')
    
    if side == 'BUY':
        # Need quote asset for buying
        required_balance = float(quantity) * float(price)
        balance = get_asset_balance(quote_asset)
        available = float(balance['data']['available'])
        
        return available >= required_balance
    else:
        # Need base asset for selling
        balance = get_asset_balance(base_asset)
        available = float(balance['data']['available'])
        
        return available >= float(quantity)

# Example usage
if can_place_order('BTC-USD', 'BUY', '0.01', '45000'):
    print("Sufficient balance to place order")
else:
    print("Insufficient balance")
```

## Portfolio Summary

Calculate total portfolio value:

```python
def get_portfolio_value():
    """Calculate total portfolio value in USD."""
    
    balances = get_all_balances()
    total_value = 0
    
    for balance in balances['data']:
        asset = balance['asset']
        amount = float(balance['total'])
        
        if asset == 'USD':
            value = amount
        else:
            # Get current price for asset
            price_response = requests.get(
                f'https://api.roxom.com/v1/market/price/{asset}-USD',
                headers=headers
            )
            
            if price_response.status_code == 200:
                price = float(price_response.json()['data']['price'])
                value = amount * price
            else:
                value = 0
        
        total_value += value
        print(f"{asset}: {amount} (${value:.2f})")
    
    return total_value

portfolio_value = get_portfolio_value()
print(f"Total Portfolio Value: ${portfolio_value:.2f}")
```

## Best Practices

<AccordionGroup>
  <Accordion title="Balance Monitoring">
    - **Check balances** before placing orders to avoid insufficient balance errors
    - **Monitor locked balances** to understand fund availability
    - **Use WebSocket streams** for real-time balance updates
    - **Set up alerts** for low balance conditions
  </Accordion>

  <Accordion title="Risk Management">
    - **Reserve emergency funds**: Don't trade with 100% of balance
    - **Track portfolio allocation**: Monitor asset distribution
    - **Consider correlation**: Diversify across uncorrelated assets
    - **Set position limits**: Maximum percentage per asset
  </Accordion>

  <Accordion title="Technical Considerations">
    - **Cache balance data** appropriately to reduce API calls
    - **Handle precision**: Use decimal libraries for financial calculations
    - **Account for fees**: Factor in trading fees when calculating available funds
    - **Sync across systems**: Ensure balance consistency across your applications
  </Accordion>
</AccordionGroup>
