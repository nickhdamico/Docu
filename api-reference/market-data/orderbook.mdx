## Get Order Book

Retrieve current order book data for a trading pair showing current bids and asks.

### Endpoint
```http
GET /v1/market/orderbook/{symbol}
```

### Parameters

<ParamField path="symbol" type="string" required>
  Trading pair symbol (e.g., "BTC-USD")
</ParamField>

<ParamField query="depth" type="integer" default="20">
  Number of price levels to return. Range: 1-100
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_KEY`
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X GET "https://api.roxom.com/v1/market/orderbook/BTC-USD?depth=10" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.roxom.com/v1/market/orderbook/BTC-USD',
    headers=headers,
    params={'depth': 10}
)

orderbook = response.json()
print(f"Best bid: {orderbook['data']['bids'][0][0]}")
print(f"Best ask: {orderbook['data']['asks'][0][0]}")
```

```javascript JavaScript
const response = await fetch('https://api.roxom.com/v1/market/orderbook/BTC-USD?depth=10', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const orderbook = await response.json();
console.log('Best bid:', orderbook.data.bids[0][0]);
console.log('Best ask:', orderbook.data.asks[0][0]);
```
</RequestExample>

<ResponseField name="data" type="object">
  Order book data for the requested trading pair

  <Expandable title="Order book properties">
    <ResponseField name="symbol" type="string">
      Trading pair symbol that was requested
    </ResponseField>
    
    <ResponseField name="bids" type="array">
      Array of bid orders [price, quantity] sorted by price descending
    </ResponseField>
    
    <ResponseField name="asks" type="array">
      Array of ask orders [price, quantity] sorted by price ascending
    </ResponseField>
    
    <ResponseField name="timestamp" type="string">
      ISO 8601 timestamp when order book snapshot was taken
    </ResponseField>
  </Expandable>
</ResponseField>

## Understanding Order Books

<AccordionGroup>
  <Accordion title="Bid vs Ask">
    - **Bids**: Buy orders showing the price buyers are willing to pay
    - **Asks**: Sell orders showing the price sellers are asking for
    - **Spread**: The difference between the highest bid and lowest ask
  </Accordion>

  <Accordion title="Price-Time Priority">
    Orders are sorted by:
    1. **Price priority**: Best prices first (highest bids, lowest asks)
    2. **Time priority**: Earlier orders at the same price level come first
  </Accordion>

  <Accordion title="Market Impact">
    Use order book data to:
    - Estimate market impact of large orders
    - Find optimal execution prices
    - Assess market liquidity and depth
  </Accordion>
</AccordionGroup>
