## Overview

The Account API provides access to your account information, including balances, trading fees, and account status. All endpoints require authentication.

## Get Account Information

Retrieve general account information and status.

### Endpoint
```http
GET /v1/account
```

### Parameters

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_KEY`
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X GET "https://api.roxom.io/v1/account" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.roxom.io/v1/account',
    headers=headers
)

account_data = response.json()
print(f"Account ID: {account_data['data']['account_id']}")
```

```javascript JavaScript
const response = await fetch('https://api.roxom.io/v1/account', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const accountData = await response.json();
console.log('Account status:', accountData.data.status);
```
</RequestExample>

## Get Account Balances

Retrieve current balances for all assets.

### Endpoint
```http
GET /v1/account/balances
```

### Parameters

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_KEY`
</ParamField>

<ParamField query="asset" type="string">
  Filter by specific asset (e.g., "BTC")
</ParamField>

<ParamField query="non_zero_only" type="boolean" default="false">
  Only return assets with non-zero balances
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X GET "https://api.roxom.io/v1/account/balances?non_zero_only=true" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```
</RequestExample>

### Response

<ResponseExample>
```json Success
{
  "data": [
    {
      "asset": "USD",
      "total": "10000.00000000",

      "locked": "500.00000000"

      "asset": "BTC",
      "total": "0.50000000",
      "available": "0.45000000",
      "locked": "0.05000000"

    {
      "asset": "ETH",
```

## Get Account Activity

Retrieve account activity history including deposits, withdrawals, and trades.

### Endpoint
```http
GET /v1/account/activity
```

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `type` | string | No | Filter by activity type: "deposit", "withdrawal", "trade", "fee" |
| `asset` | string | No | Filter by asset |
| `start_time` | integer | No | Start timestamp (Unix milliseconds) |
| `end_time` | integer | No | End timestamp (Unix milliseconds) |
| `limit` | integer | No | Number of records (default: 100, max: 500) |
| `cursor` | string | No | Pagination cursor |

### Example Request

```bash
curl -X GET "https://api.roxom.io/v1/account/activity?type=trade&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

## Account Status Values

| Status | Description |
|--------|-------------|
| `active` | Account is fully operational |
| `pending` | Account is pending verification |
| `restricted` | Account has trading restrictions |
| `suspended` | Account is temporarily suspended |
| `closed` | Account is permanently closed |

## Account Types

| Type | Description |
|------|-------------|
| `individual` | Personal trading account |
| `corporate` | Business/institutional account |
| `pro` | Professional trading account |

## Permissions

| Permission | Description |
|------------|-------------|
| `trading` | Can place and cancel orders |
| `market_data` | Can access market data |
| `account_read` | Can view account information |
| `withdrawal` | Can withdraw funds |
| `deposit` | Can deposit funds |

## Balance Breakdown

Understanding your balance components:

### Total Balance
The complete amount of an asset in your account.

### Available Balance  
The amount available for trading and withdrawals.

### Locked Balance
The amount currently locked in open orders or pending transactions.

```
Total Balance = Available Balance + Locked Balance
```

## Error Handling

Common error scenarios:

### Insufficient Permissions

```json
{
  "error": {
    "code": 403,
    "message": "Insufficient permissions",
    "details": "API key does not have account_read permission"
  }
}
```

### Asset Not Found

```json
{
  "error": {
    "code": 404,
    "message": "Asset not found",
    "details": "Asset 'INVALID' is not supported"
  }
}
```

### Account Suspended

```json
{
  "error": {
    "code": 403,
    "message": "Account suspended",
    "details": "Account is temporarily suspended. Contact support for assistance."
  }
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Balance Monitoring">
    - Check balances before placing orders to avoid insufficient balance errors
    - Monitor locked balances to understand fund availability
    - Use webhooks or WebSocket streams for real-time balance updates
  </Accordion>

  <Accordion title="Activity Tracking">
    - Regularly review account activity for security monitoring
    - Use appropriate time ranges to avoid large data sets
    - Implement pagination for historical data retrieval
  </Accordion>

  <Accordion title="Fee Optimization">
    - Monitor your trading volume to qualify for fee reductions
    - Use maker orders when possible to benefit from lower fees
    - Track fee costs to optimize trading strategies
  </Accordion>
</AccordionGroup>
