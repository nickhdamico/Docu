## Overview

The Roxom REST API provides programmatic access to all trading functionality including market data, order management, account information, and position tracking.

### Base URL
```
https://api.roxom.com/v1
```

### API Characteristics

- **REST Architecture**: Standard HTTP methods (GET, POST, PUT, DELETE)
- **JSON Format**: All requests and responses use JSON
- **Rate Limited**: Subject to rate limits based on your API key tier
- **Versioned**: Current version is v1, included in the URL path

## Authentication

All API endpoints require authentication using Bearer tokens in the Authorization header.

```bash
curl -X GET "https://api.roxom.com/v1/account" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

## Response Format

All API responses follow a consistent format:

### Successful Response
```json
{
  "data": {
    // Response payload
  },
  "timestamp": "2024-01-15T10:30:00Z",
  "request_id": "req_abc123"
}
```

### Error Response
```json
{
  "error": {
    "code": 400,
    "message": "Invalid request parameter",
    "details": "The 'symbol' parameter is required"
  },
  "timestamp": "2024-01-15T10:30:00Z",
  "request_id": "req_abc123"
}
```

## Rate Limits

API rate limits are enforced per API key:

| Tier | Requests per Second | Requests per Minute |
|------|-------------------|-------------------|
| **Basic** | 10 | 600 |
| **Pro** | 50 | 3,000 |
| **Enterprise** | 200 | 12,000 |

### Rate Limit Headers

All responses include rate limit information:

```http
X-RateLimit-Limit: 50
X-RateLimit-Remaining: 47
X-RateLimit-Reset: 1642248600
```

## Error Codes

Common HTTP status codes and their meanings:

| Code | Status | Description |
|------|--------|-------------|
| `200` | OK | Request successful |
| `400` | Bad Request | Invalid request parameters |
| `401` | Unauthorized | Invalid or missing API key |
| `403` | Forbidden | API key lacks required permissions |
| `404` | Not Found | Resource not found |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server error occurred |
| `503` | Service Unavailable | Service temporarily unavailable |

## Pagination

List endpoints support pagination using cursor-based pagination:

### Request Parameters
```http
GET /v1/orders?limit=50&cursor=eyJ0aW1lc3RhbXAiOiIyMDI0LTAxLTE1VDEwOjMwOjAwWiJ9
```

### Response
```json
{
  "data": [...],
  "pagination": {
    "next_cursor": "eyJ0aW1lc3RhbXAiOiIyMDI0LTAxLTE1VDEwOjMwOjAwWiJ9",
    "has_more": true,
    "limit": 50
  }
}
```
