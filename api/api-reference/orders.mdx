## Overview

The Orders API allows you to create, modify, cancel, and query trading orders. All order operations require authentication and are subject to risk management controls.

## Order Types

Roxom supports several order types:

| Type | Description |
|------|-------------|
| **MARKET** | Execute immediately at current market price |
| **LIMIT** | Execute only at specified price or better |
| **STOP** | Market order triggered when stop price is reached |
| **STOP_LIMIT** | Limit order triggered when stop price is reached |

## Create Order

Place a new trading order.

### Endpoint
```http
POST /v1/orders
```

### Parameters

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_KEY`
</ParamField>

### Request Body

<ParamField body="symbol" type="string" required>
  Trading pair symbol (e.g., "BTC-USD")
</ParamField>

<ParamField body="side" type="string" required>
  Order side: "BUY" or "SELL"
</ParamField>

<ParamField body="type" type="string" required>
  Order type: "MARKET", "LIMIT", "STOP", "STOP_LIMIT"
</ParamField>

<ParamField body="quantity" type="string" required>
  Order quantity with 8 decimal precision
</ParamField>

<ParamField body="price" type="string">
  Order price (required for LIMIT and STOP_LIMIT orders)
</ParamField>

<ParamField body="stop_price" type="string">
  Stop trigger price (required for STOP and STOP_LIMIT orders)
</ParamField>

<ParamField body="time_in_force" type="string" default="GTC">
  Time in force: "GTC" (Good Till Canceled), "IOC" (Immediate or Cancel), "FOK" (Fill or Kill)
</ParamField>

<ParamField body="client_order_id" type="string">
  Client-specified order identifier for tracking
</ParamField>

### Example Requests

<RequestExample>
```bash Market Order
curl -X POST "https://api.roxom.com/v1/orders" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "BTC-USD",
    "side": "BUY",
    "type": "MARKET",
    "quantity": "0.01000000"
  }'
```

```bash Limit Order
curl -X POST "https://api.roxom.com/v1/orders" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "BTC-USD",
    "side": "BUY",
    "type": "LIMIT",
    "quantity": "0.01000000",
    "price": "45000.00000000",
    "time_in_force": "GTC"
  }'
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

order_data = {
    'symbol': 'BTC-USD',
    'side': 'BUY',
    'type': 'LIMIT',
    'quantity': '0.01000000',
    'price': '45000.00000000'
}

response = requests.post(
    'https://api.roxom.com/v1/orders',
    headers=headers,
    json=order_data
)

order = response.json()
print(f"Order ID: {order['data']['order_id']}")
```
</RequestExample>

## List Orders

Retrieve all orders for your account with optional filtering.

### Endpoint
```http
GET /v1/orders
```

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `symbol` | string | No | Filter by trading pair |
| `status` | string | No | Filter by status: "NEW", "FILLED", "CANCELED", etc. |
| `side` | string | No | Filter by side: "BUY" or "SELL" |
| `start_time` | integer | No | Start timestamp (Unix milliseconds) |
| `end_time` | integer | No | End timestamp (Unix milliseconds) |
| `limit` | integer | No | Number of orders (default: 100, max: 500) |
| `cursor` | string | No | Pagination cursor |

### Example Request

```bash
curl -X GET "https://api.roxom.io/v1/orders?symbol=BTC-USD&status=NEW&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

### Response


## Cancel Order

Cancel an existing order.

### Endpoint
```http
DELETE /v1/orders/{order_id}
```

### Example Request

<CodeGroup>
```bash Cancel by Order ID
curl -X DELETE "https://api.roxom.io/v1/orders/order_abc123" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
```

## Order Status Values

| Status | Description |
|--------|-------------|
| `NEW` | Order is accepted but not yet in the order book |
| `PENDING` | Order is in the order book waiting for execution |
| `PARTIALLY_FILLED` | Order is partially executed |
| `FILLED` | Order is completely executed |
| `CANCELED` | Order was canceled |
| `REJECTED` | Order was rejected due to validation errors |
| `EXPIRED` | Order expired (for time-limited orders) |

## Time in Force

| Value | Description |
|-------|-------------|
| `GTC` | Good Till Canceled - remains active until filled or canceled |
| `IOC` | Immediate or Cancel - execute immediately, cancel remainder |
| `FOK` | Fill or Kill - execute completely or cancel entirely |

```json
{
  "error": {

  }
}
```

```json
{
  "error": {
    "code": 404,
    "message": "Order not found",
    "details": "Order 'order_invalid' does not exist"
  }
}
```

## Risk Management

Orders are subject to various risk management controls:

- **Position limits**: Maximum position size per symbol
- **Order size limits**: Minimum and maximum order quantities
- **Price bands**: Orders must be within allowed price ranges
