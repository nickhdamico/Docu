## Cancel Single Order

Cancel an existing active order by order ID.

### Endpoint
```http
POST /api/v1/orders/{order_id}/cancel
```

### Parameters

<ParamField path="order_id" type="string" required>
  Order ID to cancel (system-generated or client-provided)
</ParamField>

<ParamField header="X-API-Key" type="string" required>
  API key for authentication
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X POST "https://api.roxom.com/api/v1/orders/01234567-89ab-7def-8123-456789abcdea/cancel" \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
}

order_id = "01234567-89ab-7def-8123-456789abcdea"
response = requests.post(
    f'https://api.roxom.com/api/v1/orders/{order_id}/cancel',
    headers=headers
)

result = response.json()
print(f"Order cancelled: {result['data']['success']}")
```

```javascript JavaScript
const orderId = '01234567-89ab-7def-8123-456789abcdea';
const response = await fetch(`https://api.roxom.com/api/v1/orders/${orderId}/cancel`, {
  method: 'POST',
  headers: {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
  }
});

const result = await response.json();
console.log('Order cancelled:', result.data.success);
```
</RequestExample>

## Cancel All Orders

Cancel all active orders for a symbol or all symbols.

### Endpoint
```http
POST /api/v1/orders/cancel-all
```

### Parameters

<ParamField header="X-API-Key" type="string" required>
  API key for authentication
</ParamField>

<ParamField query="symbol" type="string">
  Cancel orders for specific symbol only. Omit to cancel all orders.
</ParamField>

<ParamField query="side" type="string">
  Cancel orders for specific side only: "BUY" or "SELL"
</ParamField>

### Example Request

<RequestExample>
```bash Cancel All Orders
curl -X POST "https://api.roxom.com/api/v1/orders/cancel-all" \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
}

# Cancel all orders
response = requests.post(
    'https://api.roxom.com/api/v1/orders/cancel-all',
    headers=headers
)

result = response.json()
print(f"Cancelled all orders: {result['data']['success']}")
```

```javascript JavaScript
// Cancel all orders for account
const response = await fetch('https://api.roxom.com/api/v1/orders/cancel-all', {
  method: 'POST',
  headers: {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
  }
});

const result = await response.json();
console.log('Cancelled all orders:', result.data.success);
```
</RequestExample>

## Cancellation States

<AccordionGroup>
  <Accordion title="CANCELED">
    **Immediate Cancellation**: Order was successfully removed from the order book
    
    - Order is no longer active
    - No further executions will occur
    - Locked funds are released immediately
  </Accordion>

  <Accordion title="PENDING_CANCEL">
    **Delayed Cancellation**: Cancel request is being processed
    
    - Order may still receive fills while cancellation is pending
    - Common during high market activity
    - Status will change to "CANCELED" once confirmed
  </Accordion>

  <Accordion title="Cannot Cancel">
    **Orders that cannot be canceled**:
    
    - Already filled orders (`FILLED`)
    - Already canceled orders (`CANCELED`)
    - Market orders that are executing (`PENDING`)
    - Expired orders (`EXPIRED`)
  </Accordion>
</AccordionGroup>

## Emergency Cancellation

For urgent situations, use the "Cancel All" endpoint to quickly close all positions:

```python
def emergency_cancel_all():
    """Emergency function to cancel all active orders."""
    
    try:
        response = requests.delete(
            'https://api.roxom.com/v1/orders',
            headers=headers,
            timeout=5
        )
        
        if response.status_code == 200:
            result = response.json()
            print(f"Emergency cancel: {result['data']['total_canceled']} orders canceled")
            return True
        else:
            print(f"Emergency cancel failed: {response.status_code}")
            return False
            
    except Exception as e:
        print(f"Emergency cancel error: {e}")
        return False
```

## Best Practices

<AccordionGroup>
  <Accordion title="Order Management">
    - **Cancel stale orders**: Remove orders that are no longer relevant
    - **Use client order IDs**: Makes it easier to track and cancel specific orders
    - **Monitor partial fills**: Be aware that partially filled orders may continue executing
  </Accordion>

  <Accordion title="Risk Management">
    - **Quick response**: Cancel orders immediately when market conditions change
    - **Bulk operations**: Use bulk cancel for efficient order management
    - **Emergency procedures**: Have a plan for quickly canceling all orders
  </Accordion>

  <Accordion title="Technical Considerations">
    - **Handle PENDING_CANCEL**: Account for delayed cancellations in your logic
    - **Verify cancellation**: Check order status after cancellation requests

  </Accordion>
</AccordionGroup>
