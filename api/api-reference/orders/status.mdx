## Get Order Status

Retrieve status and execution details of a specific order.

### Endpoint
```http
GET /api/v1/orders/{order_id}
```

### Parameters

<ParamField path="order_id" type="string" required>
  Order ID to retrieve (system-generated or client-provided)
</ParamField>

<ParamField header="X-API-Key" type="string" required>
  API key for authentication. Format: `your_api_key_here`
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X GET "https://api.roxom.com/api/v1/orders/01234567-89ab-7def-8123-456789abcdea" \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json"
```

```python Python
import requests

headers = {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
}

order_id = "01234567-89ab-7def-8123-456789abcdea"
response = requests.get(
    f'https://api.roxom.com/api/v1/orders/{order_id}',
    headers=headers
)

order = response.json()
print(f"Status: {order['data']['data']['status']}")
print(f"Quantity: {order['data']['data']['qty']}")
```

```javascript JavaScript
const orderId = '01234567-89ab-7def-8123-456789abcdea';
const response = await fetch(`https://api.roxom.com/api/v1/orders/${orderId}`, {
  headers: {
    'X-API-Key': 'your_api_key_here',
    'Content-Type': 'application/json'
  }
});

const order = await response.json();
console.log('Status:', order.data.data.status);
console.log('Quantity:', order.data.data.qty);
```
</RequestExample>

## Order Status Values

<AccordionGroup>
  <Accordion title="Active Statuses">
    **pendingsubmit**: Order has been submitted but not yet processed
    
    **waitingtrigger**: Order is waiting for trigger price to be reached
    
    **submitted**: Order has been submitted and is active in the order book
    
    **partiallyfilled**: Order has been partially filled
  </Accordion>

  <Accordion title="Completed Statuses">
    **filled**: Order has been completely filled
    
    **cancelled**: Order was cancelled
    
    **rejected**: Order was rejected
    
    **inactive**: Order is no longer active
  </Accordion>

  <Accordion title="Transition Statuses">
    **pendingcancel**: Cancel request has been submitted but not yet processed
    
    **partiallyfilledcancelled**: Partially filled order has been cancelled
  </Accordion>
</AccordionGroup>

## Real-time Updates

For real-time order status updates, use the WebSocket API:

```javascript
// Subscribe to order updates
ws.send(JSON.stringify({
  method: 'subscribe',
  params: {
    channel: 'orders'
  },
  id: 1
}));

// Receive order status changes
ws.onmessage = function(event) {
  const data = JSON.parse(event.data);
  if (data.channel === 'orders') {
    console.log('Order update:', data.data);
  }
};
```

## Order Lifecycle

<Steps>
<Step title="Order Submission">
  Order is submitted and receives initial validation.
  
  **Status**: NEW
</Step>

<Step title="Order Book Entry">
  Order passes validation and enters the order book.
  
  **Status**: PENDING
</Step>

<Step title="Matching Process">
  Order matching engine attempts to find counterparties.
  
  **Status**: PENDING or PARTIALLY_FILLED
</Step>

<Step title="Execution">
  Order is matched and executed (partially or fully).
  
  **Status**: PARTIALLY_FILLED or FILLED
</Step>

<Step title="Settlement">
  Balances are updated and confirmations sent.
  
  **Final Status**: FILLED, CANCELED, or EXPIRED
</Step>
</Steps>
