## Get Order Book Snapshot

Retrieve the current order book snapshot with bid and ask orders for a specific trading pair.

### Endpoint
```http
GET /api/v1/orderbook/snapshot
```

### Parameters

<ParamField query="symbol" type="string" required>
  Trading pair symbol (e.g., "SP500-BTC", "GLD-BTC", "OIL-BTC")
</ParamField>

<ParamField query="instType" type="string" required>
  Instrument type: "perpetual" or "spot"
</ParamField>

<ParamField query="levels" type="integer" default="20">
  Number of price levels to return per side (max: 100)
</ParamField>

### Example Request

<RequestExample>
```bash cURL
curl -X GET "https://api.roxom.com/api/v1/orderbook/snapshot?symbol=SP500-BTC&instType=perpetual&levels=50" \
  -H "Content-Type: application/json"
```

```python Python
import requests

response = requests.get(
    'https://api.roxom.com/api/v1/orderbook/snapshot',
    params={
        'symbol': 'SP500-BTC',
        'instType': 'perpetual',
        'levels': 50
    }
)

orderbook = response.json()
print(f"Best bid: {orderbook['data']['tradeable_pair']['bids'][0][0]}")
print(f"Best ask: {orderbook['data']['tradeable_pair']['asks'][0][0]}")
```

```javascript JavaScript
const response = await fetch('https://api.roxom.com/api/v1/orderbook/snapshot?symbol=SP500-BTC&instType=perpetual&levels=50');

const orderbook = await response.json();
console.log('Best bid:', orderbook.data.tradeable_pair.bids[0][0]);
console.log('Best ask:', orderbook.data.tradeable_pair.asks[0][0]);
```
</RequestExample>

<ResponseField name="data" type="object">
  Order book response container

  <Expandable title="Data properties">
    <ResponseField name="tradeable_pair" type="object">
      Order book data for the requested trading pair

      <Expandable title="Tradeable pair properties">
        <ResponseField name="symbol" type="string">
          Trading pair symbol that was requested
        </ResponseField>
        
        <ResponseField name="instrument_type" type="string">
          Type of instrument: "perpetual" or "spot"
        </ResponseField>
        
        <ResponseField name="bids" type="array">
          Array of bid orders [price, quantity] as strings in satoshis, sorted by price descending
        </ResponseField>
        
        <ResponseField name="asks" type="array">
          Array of ask orders [price, quantity] as strings in satoshis, sorted by price ascending
        </ResponseField>
        
        <ResponseField name="timestamp" type="integer">
          Timestamp when order book snapshot was taken (nanoseconds)
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Understanding Order Books

<AccordionGroup>
  <Accordion title="Bid vs Ask">
    - **Bids**: Buy orders showing the price buyers are willing to pay
    - **Asks**: Sell orders showing the price sellers are asking for
    - **Spread**: The difference between the highest bid and lowest ask
  </Accordion>

  <Accordion title="Price Format">
    - **Prices**: Expressed as strings in satoshis (1/100,000,000 of a BTC)
    - **Quantities**: Also expressed as strings in satoshis for precision
    - **Example**: "4500000" satoshis = 0.045 BTC
  </Accordion>

  <Accordion title="Price-Time Priority">
    Orders are sorted by:
    1. **Price priority**: Best prices first (highest bids, lowest asks)
    2. **Time priority**: Earlier orders at the same price level come first
  </Accordion>

  <Accordion title="Market Impact">
    Use order book data to:
    - Estimate market impact of large orders
    - Find optimal execution prices
    - Assess market liquidity and depth
  </Accordion>
</AccordionGroup>

- **GLD-BTC**: Gold futures denominated in Bitcoin  
- **OIL-BTC**: Oil futures denominated in Bitcoin
